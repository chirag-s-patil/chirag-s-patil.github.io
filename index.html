End-to-End ETL Pipeline (Docker • dbt • Streamlit)

A production-ready pipeline that extracts data, transforms it with dbt, and exposes dashboards through Streamlit — packaged & orchestrated with Docker.

Docker Streamlit Snowflake

We containerized the entire pipeline using `docker-compose` — a postgres staging database, dbt transformations that materialize marts into Snowflake, and a Streamlit app that reads the final mart for interactive dashboards. The setup includes data quality tests (dbt tests) and a lightweight scheduler.

Open Demo

Example docker-compose (trimmed):

version: "3.8"
services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_USER: dbt
      POSTGRES_PASSWORD: changeme
      POSTGRES_DB: staging
    volumes:
      - db_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  dbt:
    image: dbt:latest
    volumes:
      - ./dbt:/usr/app/dbt
    command: ["dbt", "run", "--profiles-dir", "/usr/app/dbt"]

  streamlit:
    image: streamlit/streamlit
    volumes:
      - ./app:/app
    command: streamlit run app/main.py
    ports:
      - "8501:8501"

volumes:
  db_data:
